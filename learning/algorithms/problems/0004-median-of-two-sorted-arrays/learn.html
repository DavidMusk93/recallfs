4. 寻找两个正序数组的中位数

Hard Array · Binary Search · Partition 题面 analysis.md

术语：median · partition · binary search · sentinel (−∞ / +∞)

学习怎么用 教育场景

本题核心只有两样：定位不变量 + 边界用哨兵统一。 先把不变量说清楚，再谈 binary search；公式只记一组。 测验统一提交；全对剪贴板含 #题号 · slug · 用时。

  • 自学：本质 → 不变量 → 二分方向 → 边界 → 手推 → 测。
  • 复习：口头复述「数量钉死 j + 交叉两条」与「±∞ 消特判」。
  • 卡关：先看「为什么只需交叉比较」和「i=0 怎么写」。

本质：中位数 = 找合法 partition 先立靶

不要先想「怎么 merge」。已排序的两数组，中位数只要求： 把全体切成左半 L 与右半 R，使 L 里所有数 ≤ R 里所有数， 且 |L| 正确。切点找到了，median 就在切缝两侧。

只记这一组公式

// 1) 数量不变量：左半装几个（奇数时中位数落在左半） half = (m + n + 1) // 2 j = half − i // A 左取 i 个 ⇒ B 左必须取 j 个 // 2) 顺序不变量（交叉两条即可，见下节） A[i−1] ≤ B[j] ∧ B[j−1] ≤ A[i] // 3) 答案 odd → max(A[i−1], B[j−1]) even → avg( max(A[i−1], B[j−1]) , min(A[i], B[j]) ) // 4) 边界：缺的一边用哨兵，公式不变 i=0 → A[i−1] = −∞ i=m → A[i] = +∞ j=0 → B[j−1] = −∞ j=n → B[j] = +∞
为什么 half 要 +1？ 统一「下中位数」落在左半：总长奇数时 |L| = |R|+1，median = max(L)； 偶数时 |L|=|R|，median = avg(max(L), min(R))。搜索过程奇偶相同，只返回值不同。

这题在现实里像什么 应用场景

  • 两本已排序成绩册，求全体中位数——不合并成第三本大表。
  • 两条有序报价带，求合并后的中位价。
  • 两路有序索引上的 order-statistic（第 k 小 / 中位数）。
  • 分布式两段有序数据的中位估计（partition 思想）。

二分不是炫技：题面要求 log 级。Merge 正确但 O(m+n)。 划分法用的是中位数的统计定义（均匀切半），把问题变成「找合法切分点」。

多种解法 · 对比 先选武器

A. Merge 双指针（基线）
Time O(m+n) Space O(1)

归并推进到中位下标。好写、好测，不满足 log 要求。

B. Binary partition（主推）
Time O(log min(m,n)) Space O(1)

在较短数组上二分左半个数 i； 数量不变量钉死 j = half − i； 用顺序不变量判合法 / 调方向。

在优化什么：不合并，只定位切缝。

C. 通用「第 k 小」二分
Time O(log(m+n)) Space O(1)

找第 k=(m+n+1)/2 与（偶数时）下一个。与划分同族，写法更「淘汰一半」。

解法换到了什么丢掉了什么
Merge直观正确log 复杂度
Partition 二分log、贴定义不变量 + 边界要记准
第 k 小可推广任意 k实现略绕

定位不变量：数量钉死 + 交叉比较 核心

1

数量不变量——你只有一个自由度

左半总共要 half 个。在 A 上取 i 个，B 上就必须取 j = half − i 个。j 不是搜出来的，是算出来的。 所以 binary search 的搜索空间只有 i ∈ [0, m]（并保证 m ≤ n，使 j 合法）。

2

顺序不变量——max(L) ≤ min(R)

合并后要整体有序，切缝两侧必须「左不大于右」。 因为 A、B 各自已排序，A 内部、B 内部天然满足； 唯一可能破的是跨数组：

A 的左半最大 ≤ B 的右半最小 → A[i−1] ≤ B[j] B 的左半最大 ≤ A 的右半最小 → B[j−1] ≤ A[i] // 等价写法 maxLeft = max(A[i−1], B[j−1]) minRight = min(A[i], B[j]) valid ⇔ maxLeft ≤ minRight

交叉两条 ⇔ 一个 maxLeft ≤ minRight。记交叉更利于写 if 分支。

合法切分示意 · A=[1,3] B=[2] · half=2 · i=1 → j=1
A 1 3
B 2 +∞
L = {1, 2}，R = {3}
交叉：A[0]=1 ≤ B[1]=+∞ 且 B[0]=2 ≤ A[1]=3 ✓
奇数 → median = maxLeft = max(1,2) = 2.0
偶数总长 · A=[1,2] B=[3,4] · half=2 · i=2 → j=0
A 1 2 +∞
B −∞ 3 4
L = {1,2}，R = {3,4}
交叉：2 ≤ 3 且 −∞ ≤ +∞ ✓
偶数 → median = (maxLeft + minRight)/2 = (2+3)/2 = 2.5

Binary search 方向：两条违规各自单调 怎么搜

合法条件是「且」：任一交叉失败就说明 i 偏了，且方向确定：

看到什么含义动作
A[i−1] > B[j] A 左半太大 → i 太大 缩小 i（right = i−1）
B[j−1] > A[i] A 左半太小 → i 太小 增大 i（left = i+1）
两条都 ≤ 合法 partition 停，算 median
不要同时乱调。实现里通常先判「i 太大」，再判「i 太小」，else 即命中。 在较短数组上二分，保证 j 始终落在 [0, n]。

边界 case：哨兵统一，不是一堆 if 边界

Hard 点常卡在「A 左半为空 / 取光 / B 同理」。 正确姿势：缺的一侧当成 −∞ 或 +∞，交叉公式一字不改。

Aleft = (i == 0) ? −∞ : A[i−1] Aright = (i == m) ? +∞ : A[i] Bleft = (j == 0) ? −∞ : B[j−1] Bright = (j == n) ? +∞ : B[j] // 比较与返回值仍用同一套
i = 0 · A 不贡献左半
A −∞ …
Aleft=−∞ → Aleft ≤ Bright 永真； 只靠 B 侧约束 i。
i = m · A 全部进左半
A … +∞
Aright=+∞ → Bleft ≤ Aright 永真； 另一侧同理。
  • 一数组为空 → 退化为单数组中位数（另一侧 i 或 j 走满/走空）。
  • 短数组二分：先 if m > n { swap }，避免 j 越界。
  • 奇偶只改返回，不改搜索与哨兵。

手推一遍（对着不变量） 巩固

例 1 · 奇数  A=[1,3]  B=[2]   m=2 n=1 → 可先保证短的在前
  half = (2+1+1)//2 = 2
  试 i=1 → j=1
    Aleft=1, Aright=3, Bleft=2, Bright=+∞
    1≤+∞ 且 2≤3  → 合法

  half=2；短数组 A 上二分 i∈[0,2]
  试 i=1 → j=1
    Aleft=1, Aright=2, Bleft=3, Bright=4
    1≤4 但 3≤2? 否 → Bleft > Aright → i 太小

  （实现时 m=0，i 只能 0，j=half）

发散：约束一变 思维

  • 不要求 log → Merge 足够，面试可先写对再优化。
  • 求第 k 小（任意 k）→ 同一「数量 + 淘汰」思想，half 换成 k。
  • 三个有序数组中位数 → 多路划分更难，或换 selection 模型。
  • 数组未排序 → 先排序 O((m+n)log) 或线性选择（期望 / 最坏）。

Q1. 本题最优路径的「定位目标」是？

解析：中位数 = 均匀切半后的切缝；不合并也能定位。

Q2. 在 A 上取左半 i 个后，B 的左半个数 j 由什么决定？

解析：左半总数固定；j 被 i 钉死，搜索自由度只有 1。

Q3. 顺序不变量「交叉两条」是？（多选）

解析：A、B 各自有序，内部不用再判；只判跨数组交叉。

Q4. 若 A[i−1] > B[j]，应如何调？

解析：A 左半偏大，说明从 A 拿多了，减小 i。

Q5. i=0（A 左半空）时，Aleft 应视为？

解析：端点用 ±∞ 哨兵，交叉比较无需额外分支语义。

Q6. nums1=[1,2], nums2=[3,4] 的 median 是？

解析：合并 1,2,3,4 偶数 → (2+3)/2 = 2.5；合法划分 maxLeft=2, minRight=3。

Q7. 总长为奇数时，median 取？

解析：half=(m+n+1)//2 把下中位数放进左半 → 奇数答案即 maxLeft。
