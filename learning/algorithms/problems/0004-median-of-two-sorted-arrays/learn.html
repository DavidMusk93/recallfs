4. 寻找两个正序数组的中位数

术语：median · partition · binary search · maxLeft / minRight

这题在现实里像什么 应用场景

  • 两所学校已排序的成绩表，快速求全体中位数（不合并大表）。
  • 两条有序价位带，求合并后的中位成交价。
  • 分布式两路有序流的中位估计（教学类比 partition）。
  • 数据库两段有序索引上的 order-statistic 查询。

二分不是炫技：题目明确要 log 级。Merge 正确但 O(m+n)。 划分法用「中位数切一半」的统计定义，把问题变成「找合法切分点」。

多种解法 · 发散思维 对比

A. Merge 双指针（基线）
Time O(m+n) Space O(1)

像归并，只推进到中位下标。好写、好测，但不满足 log 要求。

B. Binary partition（主推）
Time O(log min(m,n)) Space O(1)

在较短数组上二分左半个数 i， j = (m+n+1)/2 - i。 合法当且仅当 Aleft ≤ Bright 且 Bleft ≤ Aright。

在优化什么：不合并，只定位「第 k 小边界」。

C. 通用「第 k 小」二分
Time O(log(m+n)) Space O(1)

找第 k=(m+n+1)/2 与（偶数时）下一个。与划分同族，写法更「选 k」。

解法换到了什么丢掉了什么
Merge直观log 复杂度
Partition 二分log、贴题意边界要小心
第 k 小可推广实现略绕

例：A=[1,3] B=[2]  m=2,n=1 → 先保证短的在前：可交换
half = (2+1+1)/2 = 2
试 i=1 → j=1：左 {1 | 2} 右 {3 | …}
maxLeft=max(1,2)=2，minRight=3 → 2≤3 合法
奇数总长 → median = 2.0

划分条件怎么记 核心

  • 数量：左半共 (m+n+1)/2 个（奇数时多一个落左半）。
  • 顺序：左半最大值 ≤ 右半最小值 → 拆成交叉两条： A[i-1] ≤ B[j] 且 B[j-1] ≤ A[i]。
  • 哨兵：i=0 时 A 左边当 −∞；i=m 时 A 右边当 +∞（B 同理）。
  • 返回：奇数 maxLeft；偶数 (maxLeft+minRight)/2.0。
若 A[i-1] > B[j]  → i 太大，右边界左移
若 B[j-1] > A[i]  → i 太小，左边界右移
否则              → 找到划分

发散：约束一变 思维

  • 不要求 log → Merge 足够，面试先写对再优化。
  • 求第 k 小（任意 k）→ 同一套「排除一半」思想。
  • 三个有序数组中位数 → 更难，需多路划分或换模型。
  • 数组未排序 → 先排序 O((m+n)log) 或 selection 算法。

Q1. 题目对时间复杂度的核心要求是？

解析：题面要求 log 级；merge 不达标。

Q2. nums1=[1,2], nums2=[3,4] 的 median 是？

解析：合并 1,2,3,4 → 偶数 → (2+3)/2 = 2.5。

Q3. 划分法中，若在 A 上取左半 i 个，B 左半个数 j 应？

解析：左半总数固定为 half=(m+n+1)/2。

Q4. 合法划分需要满足？（多选）

解析：交叉比较两条；长度可以不同。

Q5. 为什么常在较短数组上二分 i？

解析：短数组二分更稳、更快。

Q6. 总长为奇数时，median 通常取？

解析：half 定义让中位数落在左半边界 maxLeft。

Q7. i=0（A 左半空）时，Aleft 应视为？

解析：哨兵 −∞ / +∞ 处理端点划分。
