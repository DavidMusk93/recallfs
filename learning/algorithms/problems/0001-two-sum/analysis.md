# 1. 两数之和 · 分析

## 1. 结论

| 项 | 内容 |
| --- | --- |
| 一句话题意 | 在数组里找 **两个不同下标**，使对应值之和等于 `target`，返回下标。 |
| 主推思路 | **一遍哈希**：扫到 `x` 时查是否见过 `target - x`；没有则把 `x → 下标` 记入表。 |
| 时间 / 空间 | O(n) / O(n) |
| 关联 pattern | `hashmap-complement`（补数哈希） |
| 难度 | Easy · 免费题 |
| 链接 | https://leetcode.cn/problems/two-sum/ |

题设保证 **恰好一组解**，且 **同一元素不能用两次**（同值不同下标可以，如 `[3,3]`）。

## 2. 题面形式化

### 2.1 输入 / 输出 / 约束

| 项 | 内容 |
| --- | --- |
| 输入 | `nums: Vec<i32>`，`target: i32` |
| 输出 | 两个下标的 `Vec<i32>`（顺序任意） |
| 约束 | `2 ≤ n ≤ 10⁴`，元素与 target 在 `[-1e9, 1e9]`，**仅一解** |
| Rust 签名 | `pub fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32>` |

样例：

| 输入 | 输出 | 说明 |
| --- | --- | --- |
| `nums=[2,7,11,15], target=9` | `[0,1]` | 2+7=9 |
| `nums=[3,2,4], target=6` | `[1,2]` | 2+4=6 |
| `nums=[3,3], target=6` | `[0,1]` | 同值不同下标 |

### 2.2 边界与陷阱

- 返回的是 **下标**，不是数值本身。
- 不能 `i == j`：哈希应先查补数，再写入当前值（或写入前确认不是自己）。
- 负数、零都合法：`target - x` 直接算即可。
- n 到 1e4：O(n²) 能过但 follow-up 要求更好；养成 O(n) 习惯。

## 3. 思路演进

### 3.1 暴力

```text
+------------------+
| i = 0..n-1       |
+--------+---------+
         |
         v
+------------------+
| j = i+1..n-1     |
+--------+---------+
         |
         v
+------------------+---- yes ---->+-----------+
| nums[i]+nums[j] |              | return    |
| == target ?      |              | [i, j]    |
+------------------+              +-----------+
```

- 时间 O(n²)，空间 O(1)
- 正确但无「记忆」：每次都重新扫后半段找补数

### 3.2 主推：一遍哈希（补数）

核心不变量：

> 扫到下标 `i` 时，表中存的是 **更早出现过的值 → 下标**。  
> 若 `need = target - nums[i]` 已在表中，则 `(map[need], i)` 即解。

```text
+---------------------+
| 空 HashMap          |
| value -> index      |
+----------+----------+
           |
           v
+---------------------+
| 取 x = nums[i]      |
| need = target - x   |
+----------+----------+
           |
           v
+---------------------+---- 命中 ---->+----------------+
| map 含 need ?       |               | 返回 [j, i]    |
+----------+----------+               +----------------+
           | 未命中
           v
+---------------------+
| map.insert(x, i)    |
| i += 1，继续        |
+---------------------+
```

| 对比 | 暴力 | 一遍哈希 |
| --- | --- | --- |
| 找补数 | 每次线性扫 | O(1) 均摊查表 |
| 时间 | O(n²) | O(n) |
| 空间 | O(1) | O(n) |
| 直觉 | 「两两试」 | 「我缺谁？以前见过吗？」 |

两遍哈希也可以（先全表入库再查），但 **同值两次** 时要小心；一遍先查后写更干净。

## 7. 实现提示（不含完整代码）

闸门通过前只给骨架级提示：

1. 结构：`HashMap<i32, i32>`（值 → 下标）
2. 循环：`for (i, &x) in nums.iter().enumerate()`
3. 顺序：**先** `get(&(target - x))`，**再** `insert(x, i as i32)`
4. 返回：`vec![j, i as i32]`
