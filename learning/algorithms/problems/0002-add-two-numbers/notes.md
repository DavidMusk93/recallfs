# 2. 两数相加 · notes

## 一句话

逆序链表加法 = 竖式：`s = a + b + carry`，写 `s%10`，`carry = s/10`；循环条件含最终 carry。

## 实现要点（Rust）

| 点 | 做法 |
| --- | --- |
| 头节点 | dummy head，返回 `dummy.next` |
| 短链 | `as_ref().map(\|n\| n.val).unwrap_or(0)` |
| 前进 | `l1 = l1.and_then(\|n\| n.next)` |
| 尾指针 | `tail` 始终指已写完的最后一节 |

## 易错

1. 两链都空了但 `carry == 1` → 还要挂节点（`5+5`）。  
2. 不要先转 `i128`：位数到 100。  
3. 输入已是 reverse order，**不要**先 reverse。

## 理解测备忘（本次会话）

- 手推输出 `[7,0,8]` 时注意用英文逗号。  
- 图解/不等长/尾 carry 值得对照实现。

## 性能怎么读（5 ms 才 4%？）

| 事实 | 含义 |
| --- | --- |
| 复杂度已最优 | 每位至少写一个节点 → Θ(L) 次 `Box` 分配，L=max(m,n)≤100 |
| 榜上「击败 x%」噪声极大 | 同算法重提常在 1–8 ms 之间跳；4% 与 80% 差的可能只是调度 |
| 内存 2.25 MB / ~70% | 正常：链表结果本身就要堆分配 |
| 不值得追求的「优化」 | 转大整数、递归、预处理反转——常数更差或偏题 |

可做的**常数**微调（不改变复杂度，本地已写）：

1. dummy 放栈上，少一次无用堆分配  
2. `if let Some(node) = l1 { … l1 = node.next }` 一次 move，代替 `as_ref` + `and_then`  

**结论**：AC + 清晰 carry 循环即可；别为百分位去把代码写脏。重提 1～2 次若落到 1–2 ms 属正常波动。
