# 2. 两数相加 · 分析

## 1. 结论

| 项 | 内容 |
| --- | --- |
| 一句话题意 | 两个 **逆序** 存数字的链表做加法，返回同样逆序的和链表。 |
| 主推思路 | **同步扫描 + 进位 carry**：每位 `sum = a + b + carry`，当前位 `sum % 10`，进位 `sum / 10`。 |
| 时间 / 空间 | O(max(m,n)) / O(max(m,n))（输出链；若不算输出则 O(1) 辅助） |
| 关联 pattern | `linked-list-carry`（链表进位模拟） |
| 难度 | Medium · 免费 |
| 链接 | https://leetcode.cn/problems/add-two-numbers/ |

关键：链表是 **低位在前**（个位是 head），正好符合从左到右加的习惯，不必先反转。

## 2. 题面形式化

### 2.1 输入 / 输出 / 约束

| 项 | 内容 |
| --- | --- |
| 输入 | `l1`, `l2`：非空单链表，每位 0–9，逆序表示非负整数 |
| 输出 | 同样逆序的和链表 |
| 长度 | 各链表节点数 ∈ [1, 100] |
| 保证 | 除 0 外无前导零（值语义上） |
| Rust | `add_two_numbers(l1: Option<Box<ListNode>>, l2: Option<Box<ListNode>>) -> Option<Box<ListNode>>` |

样例：

| l1 | l2 | 含义 | 输出 | 含义 |
| --- | --- | --- | --- | --- |
| `[2,4,3]` | `[5,6,4]` | 342+465 | `[7,0,8]` | 807 |
| `[0]` | `[0]` | 0+0 | `[0]` | 0 |
| `[9,9,9,9,9,9,9]` | `[9,9,9,9]` | 很长进位 | `[8,9,9,9,0,0,0,1]` | 注意最后多出一位 1 |

### 2.2 边界与陷阱

- **长度不同**：短的当 0，继续扫长的。
- **最终进位**：两链都空但 `carry==1` 时还要再挂一个节点。
- **不要**先转成整数：长度到 100 位会爆 `i128` 也不稳妥；题意就是按位模拟。
- Rust：`Option<Box<ListNode>>` 所有权——常用 dummy head + `tail` 指针写法。

## 3. 思路演进

### 3.1 错误直觉

```text
链表 → 转成整数 → 相加 → 再拆回链表
```

- 位数 ≤ 100 时大整数麻烦；且偏离本题考点。

### 3.2 主推：按位 + carry

```text
+---------------------------+
| carry = 0                 |
| dummy head, tail 指向 dummy|
+-------------+-------------+
              |
              v
+---------------------------+
| 当 l1 或 l2 或 carry 还有  |
+-------------+-------------+
              |
              v
+---------------------------+
| a = l1.val 或 0           |
| b = l2.val 或 0           |
| s = a + b + carry         |
| 新节点 val = s % 10       |
| carry = s / 10            |
| l1/l2 前进（若有）         |
+-------------+-------------+
              |
              v
+---------------------------+
| 返回 dummy.next           |
+---------------------------+
```

| 对比 | 转整数 | 按位模拟 |
| --- | --- | --- |
| 正确性 | 大数麻烦 | 自然处理任意长度 |
| 复杂度 | 仍 O(n) 但实现重 | O(n) 清晰 |
| 考点 | 偏编码转换 | 链表 + 进位 |
