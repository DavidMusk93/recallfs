这题在现实里像什么 应用场景

  • 最长「没有重复商品」的连续下单片段（促销规则）。
  • 阅读进度条：最长不重复章节浏览区间。
  • 键盘连击检测：最长不重复按键序列的长度。
  • 日志流上最长无重复 token 的窗口统计。
  • 协议解析：连续字段中 unique key 的最大跨度。
  • 流式去重窗口的长度上界（教学类比）。

Sliding window 不是为了「看起来高级」。当答案是连续区间、 且区间合法性可用两端指针单调维护时，用窗口把 O(n²) 检查压到 O(n)。 若要的是 subsequence，模型就完全不同。

多种解法 · 发散思维 对比

A. Brute force
Time O(n²)~O(n³) Space O(Σ)

枚举所有 substring，用 set 判重。建立「合法」定义，n 大时不可用。

B. Sliding window + set
Time O(n) Space O(Σ)

right 扩张；窗口内重复则 left 一步步右移并删除字符。正确，但 left 可能多次挪。

C. Sliding window + last index（主推）
Time O(n) Space O(Σ)

HashMap 记 char → last index。 若 last[c] ≥ left，则 left = last[c] + 1； 再更新 last[c]=right 与 max 长度。

在优化什么：重复时 left 一次跳到安全位置，而不是一步步挪。

解法换到了什么丢掉了什么
Brute定义清晰时间
Window + set线性均摊left 逐步移动
last indexleft 一次跳到位要正确用 max(left, …)

为什么是 left = max(left, last[c]+1)？ 卡点 · 小黑图解

窗口 s[left ..= right] 内字符必须互异。 last[c]+1 = 越过「上一次 c」； max = left 只增不减，过期的 last 不能把 left 往回拉。

1 / 3 
手推 abba（对照上图）
right=2 'b': last[b]=1 ≥ left → left=max(0,2)=2  窗 "b"
right=3 'a': last[a]=0 < left → left=max(2,1)=2  窗 "ba"  ✓
错写法 left=last[a]+1=1 → 窗 "bba"  ✗

等价：if last[c] >= left { left = last[c] + 1; }

发散：约束一变 思维

  • 要的是 subsequence → 不是本题；模型变。
  • 至多 K 个不同字符 → 窗口 + 计数，家族题。
  • 最小覆盖子串 → 窗口收缩条件变「覆盖」而非「唯一」。
  • 字符集极小（如只有小写）→ 可用数组代替 HashMap。

Q1. 本题要求的是？

解析：必须是连续 substring。

Q2. s = "pwwkew" 的答案是？

解析：最长为 wke（或 kew）长度 3；pwke 不连续。

Q3. 主推 last-index 写法，遇到重复字符 c 时 left 如何更新？

解析：跳到上次 c 的右侧，且 left 单调不减。

Q4. 哪些说法正确？（多选）

解析：A、C 对；B 是 subsequence，错。

Q5. s = "bbbbb" 的答案是？

解析：全相同，最长无重复 substring 长度 1。

Q6. 为何 left 不能随便往左移？

解析：单调 left 是滑动窗口均摊线性的关键。

Q7. last index 写法相对「set + 逐步 left++」主要优化了？

解析：仍是 O(n)；优化的是 left 移动方式与实现常数/简洁性。
