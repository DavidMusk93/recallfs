# hashmap-complement · 补数哈希

## 1. 结论

| 项 | 内容 |
| --- | --- |
| 何时用 | 在集合里找 **与当前元素配对** 满足二元关系（和/差/异或等）的另一元素 |
| 不变量 | 结构中只保留 **已经扫过** 的元素；对当前 `x` 查询 `f(x)`（补数） |
| 复杂度 | 一遍扫描通常 **O(n) 时间 / O(n) 空间**（均摊 O(1) 查表） |
| 顺序口诀 | **先查后写**（避免同一元素用两次） |

## 2. 骨架

```text
+---------------------+
| 空 map: val -> idx  |
+----------+----------+
           |
           v
+---------------------+
| x = a[i]            |
| need = f(x)         |   e.g. target - x
+----------+----------+
           |
           v
+---------------------+---- hit ---->+-----------+
| map 含 need?        |              | 配对成功  |
+----------+----------+              +-----------+
           | miss
           v
+---------------------+
| map.insert(x, i)    |
+---------------------+
```

## 4. 常见变形与坑

| 变形 / 坑 | 处理 |
| --- | --- |
| 返回值还是下标 | 表存 index，不是只存 bool |
| 同一元素用两次 | 先查后写；或两遍哈希时注意 count |
| 多解 / 计数 | 改存频次或列表，题意不同 |
| 排序 + 双指针 | 若只需值、可破坏下标时可选 |
| 负数 / 零 | 哈希键直接用数值即可 |
