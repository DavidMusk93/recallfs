# Tool Usage

目标：让 agent 可以主动补齐工具链，同时避免污染系统环境。

## 守则

- 缺少命令行工具时，可以直接安装必要依赖，例如 `ebpf`、`tmux`、`jq`、`ripgrep`。
- 安装前先判断是否已有等价工具，优先复用现有能力。
- Python 工具必须使用 `virtualenv` 或项目内 `.venv` 管理依赖。
- 不把实验性依赖安装到系统 Python 或全局环境。
- 长周期任务优先使用 `tmux`、日志文件或后台进程，避免会话中断丢失状态。
- 工具安装、环境变量、关键命令要记录到文档或脚本，方便复现。

## Python 环境

```text
+--------------------+
| 需要 Python 依赖?   |

          |

+--------------------+              +--------------------+
          | no

+---------+----------+
          |
          v

+---------+----------+
          |
          v
+--------------------+

+--------------------+
```

## 安装边界

- 可以安装：诊断、构建、测试、性能分析、格式转换所需工具。
- 谨慎安装：常驻服务、数据库、中间件、会修改系统配置的工具。
- 避免安装：来历不明、需要高权限且无法解释用途的工具。

## Agent 提醒

- 工具是为了推进任务，不是为了堆环境。
- 先用最小工具完成闭环，再补更复杂的能力。
- 每次安装都应服务于当前目标，并留下可复现路径。
