# Lesson：本地 HTML 配图「白块 / 很慢」——根因，不是压缩

> 事件：2026-07-15 · `projects/s-ui/docs/index.html`  
> 原则：**先找到原因，再改设计；规避手段不能替代根因。**

---

## 1. 现象

| 用户看到的 | 当时页面上的事实 |
| --- | --- |
| 配图渲染有问题 | 画廊里 01–03 可见，04–05 大块白区，说明有文案/框，像图没出来 |
| 特别慢 | 底部一次堆 5 张 1280×720 JPEG；滚动进场时 decode 扎堆 |
| 配图不跟主题 | 所有图集中在「概念配图」一节，读概念时看不到图 |

---

## 2. 错误路径（规避，没进核心）

```text
看到「慢 / 白」
  → 默认当成「体积太大 / 网络带宽」
  → 压缩 WebP、缩小分辨率
  → 用户纠正：这是 local page，为什么要压缩？
```

| 错误假设 | 为何不成立 |
| --- | --- |
| 本地页瓶颈 = 下载带宽 | `file://` 或本机 `http.server`，IO 是磁盘，不是 CDN 往返 |
| 先压缩总能改善体验 | 可能略减 decode 成本，但**不解释**「图与主题脱节」；也可能**掩盖**真正 bug |
| 症状消失 = 问题解决 | 压缩后图还在画廊底部 → 学习路径仍断 |

**学到的元规则：**  
症状（慢）≠ 诊断（带宽）。本地 / 远程、学习文档 / 生产站点，约束不同，默认处方不能照搬。

---

## 3. 根因分层（真正该进的核心）

### 3.1 产品结构根因（主因 · 体验）

配图被做成 **独立 Gallery**，与叙事章节解耦：

```text
读「入站/出站」→ 无图 → 滑到文末画廊才看见
→ 认知链路断裂（图是解释器，不是附录相册）
```

这不是性能 bug，是 **信息架构错误**。  
用户说「应该跟随相关主题」——直接点中这一层。

### 3.2 渲染策略根因（主因 · 白块 / 假慢）

| 机制 | 作用 | 在本地单页上的后果 |
| --- | --- | --- |
| `.reveal { opacity: 0 }` + IntersectionObserver 再 `.is-in` | 滚动入场动画 | **内容默认不可见**；IO 未触发/延迟 → 空白卡片；file:// 与滚动边界下更脆 |
| 画廊一次 5 张大图 + 同屏 decode | 进视口才集中解码 | 感知「特别慢」；未 decode 前 `aspect-ratio` 区域是白底 → 像「图坏了」 |
| `loading="lazy"` 叠在画廊末尾 | 延后加载 | 本地收益小；与「必须立刻可读」的学习页目标冲突时可帮倒忙 |

白块优先怀疑：**「看不见」是 CSS 状态，不是文件坏了**。  
慢优先怀疑：**「同时 decode 多张」+「进场才加载」**，不是「没 gzip」。

### 3.3 次因（可改善，但不是第一刀）

| 次因 | 说明 |
| --- | --- |
| 单张 100–170KB / 1280 宽 | 本地完全可接受；生产站点再谈压缩 |
| 权限 `600` | 仅 owner 可读；本机本人打开通常无影响 |
| emil 风格「stagger + opacity enter」 | 适合装饰 UI，**不适合正文必须立即可见的学习资产** |

---

## 4. 因果链（ASCII）

```text
                    ┌─ 结构：图在文末画廊 ──────────────► 主题脱节（必现）
现象「图有问题」 ──┤
                    └─ 渲染：opacity:0 / 批量 decode ──► 白块 + 假慢（易现）
                              │
                              ▼
              误诊：带宽 / 体积 ──► 压缩 ──► 未碰结构与 opacity
                              │
                              ▼
              用户纠偏：local page 不需要压缩
                              │
                              ▼
              正解：图随主题内联；内容默认可视；本地保留原图
```

---

## 5. 正确修复（对症）

| 根因 | 修复 |
| --- | --- |
| 结构脱节 | 图嵌在对应章节：管道 / 架构 / 入出站 / inbounds |
| opacity 藏内容 | 去掉「默认 opacity:0 等 IO」；学习内容**始终可见** |
| 批量画廊 | 取消独立「概念配图」导航；按需点击放大即可 |
| 本地约束 | **不压缩**原图；不为本地页引入 WebP 流水线 |

可选（非根因）：`width`/`height` 防 CLS；`decoding="async"` 降低主线程尖刺——属于打磨，不是诊断替代。

---

## 6. 可复用检查清单（以后本地 HTML 学习页）

1. **环境**：local / 内网 / 公网？瓶颈是不是带宽？  
2. **内容是否默认可视**？有没有 `opacity:0` / `hidden` / 等动画才显示？  
3. **图是否贴着第一次需要它的段落**？还是堆在文末？  
4. **「慢」是下载、解码，还是布局/动画卡顿**？分别测。  
5. **修完后能否用一句话说清根因**？说不清 = 还在规避。

---

## 7. 后续演进：AI 位图 → 内联 SVG（2026-07-15）

| 问题 | 说明 |
| --- | --- |
| 「图无法展示」 | 位图依赖相对路径 + 浏览器 decode；路径/权限/file:// 任一失败即白块 |
| AI 图不够友好 | 装饰隐喻重、标签糊、固定白底与暗色主题冲突、难改文案 |
| SVG 是否更好 | **是（对本地技术学习页）**：内联零加载、主题色、文字清晰、可贴段落 |

**演进：** 曾试 SVG 求稳；用户要小黑后改回 AI 手绘。  
**现行决策：** 主阅读用 **小黑 JPEG（贴章节）**，但必须过 §8 验收；SVG 仅作后备。

## 8. 小黑 AI 配图验收清单（2026-07-15 优化后）

| 检查 | 要求 |
| --- | --- |
| 乱码 | 无羊楷类幻觉字；印章内尽量不写字 |
| 标签位置 | 客户端在门外；数据面箭头指管道；SQLite 在控制面 |
| 词表 | 预写白名单，每词最多一次 |
| 协议名 | 贴实例：VLESS / Hysteria2 / TUIC / WS |
| 验收 | 生成后人工读完全部中文再入库 |

## 9. 一句话结晶

> **本地学习页的配图问题：根在「图与主题脱节」和「用 opacity 藏正文」；不在「没压缩」。**  
> AI 小黑图要过 **读字 + 概念箭头** 两道验收，否则比 SVG 更伤教学。
